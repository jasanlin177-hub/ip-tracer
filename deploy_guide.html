⚙️ 科偵 IP 智慧溯源系統 — 上架教學

把工具放上 Streamlit Community Cloud，產生一個公開網址，全國科偵同仁點開就能用（免費、零安裝）

這個方式的好處

項目說明
費用免費
同仁怎麼用你給一個網址，用瀏覽器點開即可，完全不用安裝任何東西
更新你改一次程式並推上 GitHub，所有人下次開就是新版
資料本工具不登入、不儲存任何查詢紀錄，只整合公開查詢

需要準備的兩個免費帳號

1GitHub 帳號：github.com/signup（放程式碼用）
2Streamlit 帳號：直接用 GitHub 帳號登入 share.streamlit.io 即可，不用另外註冊

步驟 A — 把程式放上 GitHub

1登入 GitHub，右上角 ＋ → New repository。
2Repository name 隨意（例：ip-tracer），Public / Private 皆可，按 Create repository。
3在你電腦的專案資料夾（C:\Projects\IPsearch）開啟命令列，執行： 
git init
git add .
git commit -m "科偵 IP 溯源系統"
git branch -M main
git remote add origin https://github.com/<你的帳號>/ip-tracer.git
git push -u origin main
（第一次 push 會要求登入 GitHub，照指示授權即可。）
✅ 完成後重新整理 GitHub 頁面，應該看得到 ip_analyzer.py 等檔案。

步驟 B — 在 Streamlit Cloud 上架

1用 GitHub 帳號登入 share.streamlit.io。
2點右上 Create app → 選 Deploy a public app from GitHub。
3填入：
Repository選你剛建的 <帳號>/ip-tracer
Branchmain
Main file pathip_analyzer.py
4（可選）點 Advanced settings → Python version 選 3.11 或以上。
5按 Deploy，等 2–5 分鐘安裝套件。
✅ 成功後會得到一個像 https://ip-tracer-xxxx.streamlit.app 的網址，把它貼到科偵群組即可。

需要上傳到 GitHub 的檔案

  • ip_analyzer.py（主程式）
  • tracer.py、proxy.py、batch.py、report.py（邏輯模組）
  • requirements.txt（套件清單，Streamlit 會照它安裝）
  • .streamlit/config.toml（介面主題）
  • .gitignore（已幫你排除本機測試產出）

之後要改程式怎麼更新？

git add .
git commit -m "調整說明"
git push

推上去後，Streamlit Cloud 會自動偵測並重新部署，網址不變。

⚠️ 唯一的營運提醒（不是安全問題）：
其中一個資料源 ip-api 免費版是「依伺服器 IP 限速 45 次/分」。全國共用同一個網址時大家共用這個額度， 尖峰時可能偶爾要稍等。若使用量大，可申請 ip-api 付費金鑰，或改為各單位在自己電腦本機執行（各自有額度）。
想加一道簡單密碼？（避免無關的人在網路上亂用你的額度）
Streamlit Cloud 的 App → Settings → Secrets 可設定，之後我可以幫你加一個輕量密碼閘。 但這與辦案機密無關——因為工具本來就不儲存任何資料。

⚠️ 疑難排解：部署後一直閃退（Segmentation fault）

症狀：App 有跑起來（log 出現 Uvicorn server started），但查詢或切換選項後過幾秒就崩潰， log 結尾出現 Segmentation fault 並不斷重啟。

原因：Streamlit 的表格元件 st.dataframe 在顯示資料時會把資料轉成 Arrow 格式， 這條路徑經過原生套件 pandas / pyarrow。當 Streamlit Cloud 自動裝到 最新的 pandas 3.0 時，這個轉換會踩到記憶體錯誤而整個崩掉（實測 Python 3.11 / 3.14 都會）。 不是程式邏輯壞，是底層套件版本組合不穩。

本專案已內建的解法（若你 fork 或照抄，記得保留）：

做法檔案
① 鎖 pandas 2.x，避開高風險的 3.0requirements.txt 加一行 pandas<3
② 表格改用純 HTML 顯示，完全不經 pandas/pyarrowip_analyzer.py 的 render_table()（用 st.markdown 畫表，取代 st.dataframe）
💡 給未來的你：之後若在 Streamlit Cloud 又遇到莫名 segfault，第一個先懷疑 st.dataframe / st.table（Arrow 轉換）與 pandas 版本沒鎖。把表格改純 HTML、requirements 鎖 pandas<3，通常就解了。
